ПРОБЛЕМА: LangChain конфликтует с другими пакетами при установке

РЕШЕНИЕ: Устанавливать в 2 этапа

═══════════════════════════════════════════════════════════════════

ВАРИАНТ 1: Установка В КОНТЕЙНЕРЕ (после сборки)
═══════════════════════════════════════════════════════════════════

1. Собрать образ (БЕЗ AI зависимостей):
   docker-compose up -d --build

2. Установить AI пакеты ВРУЧНУЮ:
   docker-compose exec api pip install langchain==0.1.6 langchain-core==0.1.23 langchain-community==0.0.20

3. Готово! Проверить:
   docker-compose exec api python -c "import langchain; print('✅ OK')"

═══════════════════════════════════════════════════════════════════

ВАРИАНТ 2: БЕЗ LangChain (упрощенная версия)
═══════════════════════════════════════════════════════════════════

Используйте ТОЛЬКО Ollama Client (уже работает):
- ✅ Ollama напрямую
- ✅ Все агенты работают
- ❌ Без LangChain features

Команды работают БЕЗ изменений:
   docker-compose exec api python process_articles.py 5

═══════════════════════════════════════════════════════════════════

ВАРИАНТ 3: Полная перестройка с AI (медленно)
═══════════════════════════════════════════════════════════════════

1. Раскомментировать в requirements.txt:
   # langchain==0.1.6
   # langchain-core==0.1.23
   # langchain-community==0.0.20

2. Пересобрать с нуля:
   docker-compose down
   docker-compose build --no-cache
   docker-compose up -d
